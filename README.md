# STM32-3D-Rendering-Engine
A lightweight **3D rendering engine** for **STM32F429** microcontrollers with LTDC display written in **C** using **STM32CubeIDE**, capable of real-time rendering of basic 3D primitives with lighting, hierarchical transformations, and input handling.

This engine is designed for embedded applications where users can define their scene, handle inputs, and render objects efficiently without worrying about low-level graphics operations.

## Features

- **3D Primitives:** Plane, Cube, Sphere, Cylinder, Circle, Ring, IcoSphere
- **Hierarchical Objects:** Parent-child relationships with local/global transformations
- **Lighting System:** Ambient and directional lights
- **Rendering Modes:** Wireframe or Filled
- **Backface Culling & Depth Sorting**
- **Input Handling:** Joystick and button interrupts integrated
- **Double-buffered Framebuffer:** LTDC double buffering for smooth animation
- **Time-based Updates:** `_delta` based updates for physics and animation

## Getting started 

### Prerequisites
- STM32CubeIDE installed
- STM32F429 development board (e.g., STM32F429I-DISCO)
- HAL libraries for LTDC, ADC, GPIO initialized (auto-generated by CubeIDE)
- Basic knowledge of C programming and embedded development

---

### Building the Project
- Download the latest release ZIP from the [Releases](../../releases) page.
- Extract the ZIP to a convenient folder on your computer.
- Open STM32CubeIDE and select File → Import → Existing Projects into Workspace.
- Select the extracted folder as root directory and import the project.
- Build the project (Project → Build Project).
- Connect your STM32F429 board and flash the firmware.
- Running the Project
- Once flashed, the MCU automatically runs the main() loop.

User code is written in `scene_root.c` using hooks: `OnInit()`, `OnTick()`, `OnRender()`, `OnButtonPress()`, and `OnJoystickInput()`.
The engine handles rendering, input, and timing internally; you only define scene logic and objects.

## Example program

```C
//scene_root.c

#include "scene_root.h"
#include "primitives.h"

// Scene objects
Object cube;
Object sphere;

void OnInit(void) {
    // Create a cube at origin
    CreateCube(&cube, 50, 50, 50, 0xFF00FF00); // Green cube
    
    // Create a sphere slightly above the cube
    CreateSphere(&sphere, 30, 12, 12, 0xFFFF0000); // Red sphere
    sphere.Position.Y = 60.0f;

    // Setup a directional light
    LightParams sun = { .dir = {1.0f, -1.0f, 0.5f}, .col = 0xFFFFFFFF, .a = 1.0f };
    SetLight(0, sun);
    EnableLight(0);
    EnableLighting(0.2f); // ambient
}

void OnTick(float _delta) {
    // Rotate cube around Y axis
    cube.Rotation.Y += 1.0f * _delta;

    // Rotate sphere around X axis
    sphere.Rotation.X += 1.5f * _delta;
}

void OnRender(void) {
    RenderBegin();

    PushPrimitive(&cube, (Vec3f){0,0,0}, (Vec3f){0,0,0});
    PushPrimitive(&sphere, (Vec3f){0,0,0}, (Vec3f){0,0,0});

    RenderEnd();
}
```

## Data Types

### Vec3f

#### Signature

```C
typedef struct {
	float X, Y, Z;
} Vec3f;
```

##### Fields

- `X` - world X coordinate of the vector
- `Y` - world Y coordinate of the vector
- `Z` - world Z coordinate of the vector
  
#### Usage

Represents a point or direction in 3D space.

Used throughout the renderer for:

- Vertex positions (e.g. triangle points)
- Transformations (translation, scaling, rotation)
- Direction vectors (normals, camera direction)

`Vec3f` is the core math primitive of the renderer and is typically manipulated using helper functions from `vector_math.h`.

---

### Triangle

#### Signature

```C
typedef struct {
	Vec3f a,b,c;
	uint32_t col;
	float avg_z;
} Triangle;
```

##### Fields

- `a, b, c` - points of the triangle
- `col` - color of the triangle in the format `0xAARRGGBB`
- `avg_z` - system variable. Updated each frame by the renderer. Should not be modified

#### Usage

Represents a single triangle primitive in 3D space.  
  
Used throughout the renderer as the fundamental building block for all geometry. More complex meshes are composed by combining multiple `Triangle` instances.  
  
Typical workflow:  
- Set `a`, `b`, `c` to define the triangle's vertices in world space  
- Assign `col` to control its color  
- Submit the triangle to the renderer each frame

---

### LightParams

#### Signature

```C
typedef struct {
	Vec3f dir;
	uint32_t col;
	float a;
} LightParams;
```

##### Fields

- `dir` - direction of the light in world space. A normalized vector typically pointing from the scene toward the light source.
- `col` - color of the light in ARGB format (`0xAARRGGBB`).
- `a` - intensity of the light. Multiplies the contribution of this light when computing lighting.

#### Usage

Represents a directional light used in the renderer.

- Can be assigned to one of the 8 light slots via `SetLight(LIGHT index, LightParams params)`.
- Used in lighting calculations during `RenderEnd()`.
- Commonly used for sunlight or fixed scene lights.
- Only affects triangles when lighting is enabled (`EnableLighting()`).

---

### Object

#### Signature  
  
```C  
typedef struct Object {  
Vec3f Position;  
Vec3f GlobalPosition;  
Vec3f Rotation;  
Vec3f Scale;  
Triangle* _tris;  
int _tcount;  
struct Object* Parent;  
struct Object* Children[OBJECT_MAX_CHILDREN];  
int _ccount;  
} Object;
```

##### Fields

- `Position` – local position of the object relative to its parent or the world origin if no parent.
- `GlobalPosition` – computed world-space position of the object after applying parent transforms.
- `Rotation` – local rotation of the object in Euler angles (radians).
- `Scale` – local scale of the object (currently not applied in renderer).
- `_tris` – pointer to the array of triangles that make up this object.
- `_tcount` – number of triangles in the `_tris` array.
- `Parent` – pointer to the parent `Object` in the hierarchy.
- `Children` – array of pointers to child objects (up to `OBJECT_MAX_CHILDREN`).
- `_ccount` – number of children currently attached to this object.

#### Usage

Represents a 3D object in the scene.

- Can contain any number of triangles, created with functions like `CreateCube()`, `CreateSphere()`, etc.
- Supports hierarchical transformations: position and rotation are applied relative to a parent if present.
- `PushPrimitive()` is used to submit the object and its children to the renderer.
- Use `AddChild()` to attach children to build complex models.
- `_tris` and `_tcount` are managed by creation functions and should not be manually modified unless advanced control is required.

---

### Frame

#### Signature  
  
```C  
typedef struct {  
uint32_t BackColor;  
uint32_t TextColor;  
sFONT* Font;  
char Label[5];  
uint8_t Width;  
uint8_t Height;  
} Frame;
```

##### Fields

- `BackColor` – background color of the frame (ARGB format `0xAARRGGBB`).
- `TextColor` – color used for text inside the frame.
- `Font` – pointer to the font used for the label text.
- `Label` – null-terminated string (up to 4 characters) displayed on the frame.
- `Width` – width of the frame in pixels.
- `Height` – height of the frame in pixels.

#### Usage

Represents a simple GUI frame that can be drawn on the screen.

- Initialize with `CreateFrame()` to set size, colors, font, and label.
- Draw at a specific world position using `DrawFrame()`.
- Useful for displaying labels, UI elements, or indicators in 3D scenes.
- Supports small text labels; frame rectangle and background are automatically drawn.

## Engine hooks

Scenes and behavior are user-defined in a file called `scene_root.c`, by using hooks exposed by the engine.

---

### Rendering hook

`void OnRender(void)`
  
Called once per frame to render the scene.  
  
- The user **must call `RenderBegin()` at the start before starting the scene** and **`RenderEnd()` at the end** to submit all triangles to the renderer.  
- Place all calls to `PushPrimitive()`, `Create*()` functions, or other rendering logic **between `RenderBegin()` and `RenderEnd()`**.  
- Runs in the main loop, outside interrupt context.

**Note**: You can also draw 2d primitives in the body, just make sure to call the draws after `RenderEnd()`, otherwise the 3D scene will be drawn on top. 

---

### Physics hook

`void OnTick(float _delta)`  
  
Called automatically at a fixed rate (1/30 second per tick).  
  
- `_delta` – time elapsed since the previous tick  
- Useful for updating animations, physics, or game logic at a consistent rate  
- Ensures deterministic updates independent of rendering speed

---

### Initialization hook

`void OnInit(void)`  
  
Called once at program start.  
  
- Use this to initialize objects, lights, camera, or GUI elements.  
- Similar to a “setup” function in other frameworks.

---

### Button input hook

`void OnButtonPress(void)`
  
Called whenever a button interrupt is triggered.  
  
- Handle button input logic here.  
- Invoked from NVIC (interrupt context).  
- Should be lightweight and fast; avoid long blocking operations.  

---

### Joystick input hook

`void OnJoystickInput(uint32_t* _buffer, float _delta)`
  
Called when joystick input is detected.  
  
- `_buffer` – pointer to the joystick input data
- `_delta` – same delta as in `OnTick()`, i.e., time elapsed since the previous tick  
- Invoked from NVIC (interrupt context)  
- Can be used to update camera rotation, object movement, or other input-dependent logic smoothly over time

## Rendering

The engine uses a **software rasterization pipeline**:

- Triangles are projected from 3D world space to 2D screen coordinates
- Backface culling removes triangles facing away from the camera
- Depth sorting ensures proper overlap
- Lighting is applied per triangle using directional and ambient lights
- Final triangles are pushed to the **LTDC framebuffer** via double-buffering

By default, engine caps triangle count to **512** and light sources to **8**

Rendering related functions are defined in `renderer.h`:

---
  
### Simple functions  
  
`void RenderBegin(void)`  
  
Initializes a new frame. Clears the internal triangle queue. Required to begin any 3d rendering operations.

---
  
`void RenderEnd(void)`
  
Finalizes rendering. Required to actually start rendering the triangle queue. 
  
- Computes triangle depths  
- Sorts triangles  by depth
- Draws all submitted triangles to the screen

---
  
`void PushTriangle(Triangle tri)`  
  
Adds a triangle to the render queue.  
  
- Respects back-face culling if enabled  
- Does nothing if the queue is full

---
  
`void SetCameraPos(Vec3f newpos)`  
  
Sets the camera position in world space.

---
  
`Vec3f GetCameraPos(void)`  
  
Returns the current camera position.

---
  
`void SetRenderMode(RENDER_MODE mode)`  
  
Sets the rendering mode.  
  
- `RENDER_MODE_WIREFRAME` - draw only edges  
- `RENDER_MODE_FILLED` - draw filled triangles

---

`void RenderEnable(RENDER_FLAG flags)`  
  
Enables one or more render flags.

---
  
`void RenderDisable(RENDER_FLAG flags)`  
  
Disables one or more render flags.

---
  
`uint8_t RenderIsEnabled(RENDER_FLAG flag)`  
  
Checks whether a render flag is enabled. Returns non-zero if enabled.

---

### Lighting functions

`void EnableLighting(float ambient)`  

---
  
Enables lighting calculations.  
  
- `ambient` defines base light level (0.0 – 1.0)

---
  
`void DisableLighting(void)`  
  
Disables lighting calculations.

---
  
`void EnableLight(LIGHT index)`  
  
Enables a specific light source.
  
---
  
`void DisableLight(LIGHT index)`  
  
Disables a specific light source.

---
  
`void SetLight(LIGHT index, LightParams params)`  
  
Sets parameters for a specific light.

---
  
`LightParams GetLight(LIGHT index)`  
  
Returns parameters of a specific light.

---

### Complex functions

`void ProjectFrustum(Vec3f world, int16_t *sx, int16_t *sy) ` 
  
Projects a 3D world point to 2D screen coordinates using perspective projection.  
  
- Accounts for camera position, FOV, aspect ratio, near plane, and screen center.

## Primitives

By default the engine exposes a couple predefined functions in `primitives.h` to build scenes more easily. Each primitive is defined by passing an `Object` and directly modifying tris.

---
  
### Initializers
#### Plane  

`void CreatePlane(Object* obj, float width, float length, uint32_t color)`  
  
Creates a rectangular plane centered at the origin.  
  
- Generates **2 triangles**  
- `width` and `length` define dimensions  
- `color` sets the color of all triangles  
- Allocates memory for `obj->_tris` and sets `_tcount`

Computed size: `88 Bytes`

---
  
#### Circle  
  
`void CreateCircle(Object* obj, float radius, int corners, uint32_t color)`  
  
Creates a flat circle in the XZ plane.  
  
- Divided into `corners` triangles  
- Vertices evenly spaced around the circle  
- Center at `(0,0,0)`  
- Allocates memory for `obj->_tris` and sets `_tcount`

Computed size: `corners*44 Bytes`

---
  
#### Ring  
  
`void CreateRing(Object* obj, float outer_rad, float inner_rad, int corners, uint32_t color)`  
  
Creates a flat ring (donut) in the XZ plane.  
  
- Composed of `corners*2` triangles  
- `inner_rad` and `outer_rad` define inner and outer radii  
- Allocates memory for `obj->_tris` and sets `_tcount`

Computed size: `corners*88 Bytes`

---
  
#### Cube  
  
`void CreateCube(Object* obj, float width, float length, float height, uint32_t color)`  
  
Creates a rectangular cuboid centered at the origin.  
  
- 12 triangles (6 faces × 2 triangles per face)  
- Allocates memory for `obj->_tris` and sets `_tcount`

Computed size: `528 Bytes`

---
  
#### Cylinder  
  
`void CreateCyllinder(Object* obj, float radius, float height, int corners, uint32_t color)`  
  
Creates a vertical cylinder along the Y axis.  
  
- Composed of `corners*4` triangles (top, bottom, and sides)  
- `radius` sets radius, `height` sets height  
- Allocates memory for `obj->_tris` and sets `_tcount`

Computed size: `corners*176 Bytes`

---
  
#### Sphere  
  
`void CreateSphere(Object* obj, float radius, int h_subdivs, int v_subdivs, uint32_t color)`  
  
Creates a sphere centered at the origin.  
  
- `h_subdivs` = horizontal subdivisions (around Y axis)  
- `v_subdivs` = vertical subdivisions (from pole to pole)  
- Generates triangles for each segment  
- Allocates memory for `obj->_tris` and sets `_tcount`

Computed size: `88*h_subdivs * (v_subdivs - 1 Bytes`

---
  
### Frame Functions  
  
`void CreateFrame(Frame* frm, uint8_t width, uint8_t height, uint32_t bcolor, uint32_t fcolor, sFONT* font, char* label)`  
  
Initializes a frame object with size, colors, font, and label.  
  
- `width`, `height` – dimensions of the frame  
- `bcolor` – background color (ARGB)  
- `fcolor` – text color (ARGB)  
- `font` – pointer to font used for label  
- `label` – up to 4-character label (null-terminated automatically)  
  
Allocates no extra memory; simply populates the `Frame` struct fields.

---
  
`void DrawFrame(Frame* frm, Vec3f position)`  
  
Draws the frame on the screen at a given world position.  
  
- Projects 3D world position to 2D screen coordinates using `ProjectFrustum()`  
- Draws rectangle and label on the LCD  
- Respects font size for label display  
- Uses `BackColor` and `TextColor` fields from the frame struct

---

### Object functions

`void PushPrimitive(Object* obj, Vec3f parent_pos, Vec3f parent_rot)`  

Submits a 3D object and all of its children to the renderer.  
  
- Computes **global position** and **rotation** from parent transforms  
- Applies rotation and translation to all triangles in the object  
- Calls `PushTriangle()` for each triangle  
- Recursively processes all children of the object  
  
Useful for hierarchical objects and complex models.  
  
---
  
`void AddChild(Object* parent, Object* child)`  
  
Attaches a child object to a parent object.  
  
- Prevents cycles or duplicate parent assignments  
- Updates parent’s `Children` array and child’s `Parent` pointer  
- Maximum children is limited by `OBJECT_MAX_CHILDREN`

## Vector math

Engine defines a set of 3D Vector functions in `vector_math.h` that operate on `Vec3f` types.

### Simple functions

`Vec3f Vec3add(Vec3f a, Vec3f b)`

Adds two vectors, returns the resulting vector.

---

`Vec3f Vec3sub(Vec3f a, Vec3f b)`

Subtracts vector **b** from vector **a**, returns the resulting vector.

---

`Vec3f Vec3mul(Vec3f a, Vec3f b)`

Multiplies two vectors component-wise, returns the resulting vector.

---

`Vec3f Vec3scale(Vec3f a, float b)`

Multiplies each component of vector **a** by float **b**, returns the resulting vector.

---

`float Vec3dot(Vec3f a, Vec3f b)`

Computes the dot product of two vectors. Returns the resulting value.

---

`Vec3f Vec3cross(Vec3f a, Vec3f b)`

Computes the cross product of two vectors. Returns a vector, perpendicular to both **a** and **b**.

---

`float Vec3len(Vec3f a)`

Computes the magnitute of vector **a**, returns the resulting value.

---

`Vec3f Vec3norm(Vec3f a)`

Normalizes the vector. Returns the result.

---

### Complex functions


`void RotateX(Vec3f* vec, Vec3f o, float a)`

Rotates the **vec** along X axis by angle **a** relative to a world space point **o**. Directly modifies the passed vector.

---

`void RotateY(Vec3f* vec, Vec3f o, float a)`

Rotates the **vec** along Y axis by angle **a** relative to a world space point **o**. Directly modifies the passed vector.

---

`void RotateZ(Vec3f* vec, Vec3f o, float a)`

Rotates the **vec** along Z axis by angle **a** relative to a world space point **o**. Directly modifies the passed vector.

---

`void Rotate(Vec3f* vec, Vec3f o, Vec3f a)`

Applies combined rotation along each axis by vector **a** representing the rotation. The rotation are in the following order:

```
RotateZ
RotateY
RotateX
```
